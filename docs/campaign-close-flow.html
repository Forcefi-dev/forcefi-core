ForceFi Campaign Close Flow

Token Distribution & Fee Calculation Documentation

Prerequisites

  • Campaign must not be already closed: !fundraising.campaignClosed
  • Must reach minimum threshold: totalFundraised > hardCap * minCampaignThreshold / 100
  • Must be past reclaim window: endDate + reclaimWindow <= block.timestamp
  • Only campaign owner can close

Fee Structure

Base Fee Tiers

  • Tier 1 (<1M tokens): 5%
  • Tier 2 (1M-2.5M tokens): 4%
  • Tier 3 (>2.5M tokens): 3%

Fee Distribution

  • Referral: 10% of base fee
  • Platform: 18% of base fee
  • Staking: 27% of base fee
  • Curator: 45% of base fee

Fee Notes

  • Base fee (3-5%) is split between all parties
  • Curator fee is adjusted by curator percentage
  • Total distribution equals 100% of base fee

Flow Explanation

  1. Total raised amount is first evaluated for fee tier
  2. Base fee is calculated (3-5% based on amount)
  3. Base fee is split between platform, staking, and curators
  4. Referral fee is taken separately from total amount
  5. Curator portion is distributed based on individual percentages
  6. Remaining amount goes to project owner

Example Calculation

Total Raised: 1,000,000 tokens Base Fee (4%): 40,000 tokens Distribution: - Referral (10%): 4,000 tokens - Platform (18%): 7,200 tokens - Staking (27%): 10,800 tokens - Curator (45%): 18,000 tokens To Project Owner: 960,000 tokens

⚠️ Important Notes

  • Each whitelisted token is processed separately
  • Fees are only distributed if corresponding addresses are set
  • The contract tracks actually distributed fees rather than calculated fees
  • Any unused campaign tokens are returned to the project owner
  • All calculations use safe math to prevent overflows

Error Cases & Handling

Common Closing Failures

  • "Campaign already closed" - Attempt to close an already closed campaign
  • "Not an owner of a fundraising" - Non-owner attempting to close campaign
  • "Campaign is still within reclaim window" - Trying to close before reclaim window expires
  • "Campaign didn't reach minimal threshold" - Raised amount below minimum threshold

Fee Distribution Edge Cases

  • Unset fee addresses - Fee distribution skipped for missing addresses
  • Zero curator percentage - Curator fees skipped when percentage is 0
  • Failed transfers - Contract continues with remaining distributions
  • Insufficient balance - Reverts the entire closing operation

Token Reclaim Cases

  • "Campaign has not ended" - Too early to reclaim
  • "Campaign not closed" - Must be closed before reclaiming
  • "No fees to claim" - Already claimed or no fees to claim
  • "Cannot release tokens" - Vesting conditions not met

Closing Workflow

  1. Validate closing conditions:
    • Campaign not already closed
    • Past end date + reclaim window
    • Minimum threshold reached (totalFundraised > hardCap * minCampaignThreshold)
    • Caller is campaign owner
  2. Calculate and process fees:
    • Determine fee tier based on total raised
    • Calculate base fee amount
    • Split and distribute to platform, staking, and curator
    • Process referral fee if applicable
  3. Finalize campaign:
    • Mark campaign as closed
    • Transfer remaining funds to project owner
    • Emit CampaignClosed event
  4. Post-closing actions:
    • Enable token claiming for investors
    • Allow curator fee claims
    • Start vesting schedule if configured
